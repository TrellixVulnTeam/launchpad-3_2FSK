If a product doesn't use Launchpad to track its bugs, there's
information about the product's bug tracker when adding an upstream
task.

    >>> user_browser.open(
    ...    'http://launchpad.dev/firefox/+bug/1/+add-affected-product'
    ...    '?field.product=gnome-terminal')
    >>> print user_browser.contents
    <...GNOME Terminal uses
    <a href="http://bugzilla.gnome.org/bugs">GnomeGBug GTracker</a>
    to track its bugs...

    >>> user_browser.getControl('URL') is not None
    True

If a product doesn't use Launchpad, and doesn't have a bug tracker
specified, it will simply say that it doesn't use Launchpad to track its
bugs and prompt for a URL.

    >>> user_browser.open(
    ...    'http://launchpad.dev/firefox/+bug/1/+add-affected-product'
    ...    '?field.product=thunderbird')
    >>> print user_browser.contents
    <...Mozilla Thunderbird doesn't use Launchpad to track its bugs...

    >>> user_browser.getControl('URL') is not None
    True

For products using Launchpad, there won't even be a URL widget.

    >>> user_browser.open(
    ...    'http://launchpad.dev/firefox/+bug/1/+add-affected-product'
    ...    '?field.product=evolution')

    >>> user_browser.getControl('URL')
    Traceback (most recent call last):
    ...
    LookupError: label 'URL'
