There is a list of IProduct objects that have translations and are using
Launchpad Translations officially from a IProject's translations page.

GNOME is a good example, it has products with translations.

  >>> browser.open('http://translations.launchpad.dev/gnome')
  >>> print browser.url
  http://translations.launchpad.dev/gnome

Evolution being one with translations and being used officially

  >>> evo_link = browser.getLink('Evolution')
  >>> print evo_link.url
  http://translations.launchpad.dev/evolution

  >>> browser.open('http://translations.launchpad.dev/gnome')
  >>> print browser.url
  http://translations.launchpad.dev/gnome

Netapplet is another product of GNOME project. It has translations, but it's not
using Launchpad officially, so it's not in this page.

  >>> browser.getLink('Network Applet')
  Traceback (most recent call last):
  ...
  LinkNotFoundError

Let's confirm what we just stated.

  >>> anon_browser.open('http://launchpad.dev/gnome')
  >>> print anon_browser.title
  The GNOME Project in Launchpad

Netapplet is in fact a product belonging to GNOME project.

  >>> anon_browser.getLink('NetApplet').click()
  >>> print anon_browser.title
   Network Applet in Launchpad

It's not using Launchpad Translations officially

  >>> print extract_text(
  ...   find_tag_by_id(anon_browser.contents, 'table-details'))
  Active
  ...Doesn&rsquo;t use Translations...

Anonymous users don't see the translations

  >>> anon_browser.getLink('Translations').click()
  >>> print anon_browser.title
  Translations of Network Applet in Launchpad

  >>> find_tag_by_id(
  ...     anon_browser.contents,
  ...     'portlet-obsolete-translatable-series') is None
  True

but administrators see translations disabled.

  >>> admin_browser.open(anon_browser.url)
  >>> print admin_browser.title
  Translations of Network Applet in Launchpad
  >>> print extract_text(find_tag_by_id(
  ...     admin_browser.contents,
  ...     'portlet-obsolete-translatable-series'))
  All product series with obsolete templates
  NetApplet Series: trunk

alsa-utils is a product that doesn't belong to GNOME project. It has
translations and is using Launchpad Translations officially, and it 
shouldn't appear in GNOME project translations page.

  >>> browser.open('http://translations.launchpad.dev/gnome')
  >>> browser.getLink('alsa-utils')
  Traceback (most recent call last):
  ...
  LinkNotFoundError

Let's confirm what we just stated.

  >>> browser.open('http://launchpad.dev/gnome')
  >>> print browser.url
  http://launchpad.dev/gnome

alsa-utils does not belong to GNOME project.

  >>> browser.getLink('alsa-utils')
  Traceback (most recent call last):
  ...
  LinkNotFoundError

It's using Launchpad Translations officially.

  >>> browser.open('http://launchpad.dev/alsa-utils')
  >>> print browser.url
  http://launchpad.dev/alsa-utils
  >>> print extract_text(find_tag_by_id(browser.contents, 'table-details'))
  Active
  ...Uses Translations...

And it has translations.

  >>> browser.open('http://translations.launchpad.dev/alsa-utils')
  >>> print browser.url
  http://translations.launchpad.dev/alsa-utils

  >>> alsa_utils_template = browser.getLink('View Template & All Languages...')
  >>> print alsa_utils_template.url
  http://translations.launchpad.dev/alsa-utils/trunk/+pots/alsa-utils

The translations page has also a link to the product search page to see
whether a product is registered in Launchpad.

  >>> browser.open('http://translations.launchpad.dev/gnome')
  >>> print browser.url
  http://translations.launchpad.dev/gnome
  >>> browser.getLink('registered with Launchpad').click()
  >>> print browser.url
  http://translations.launchpad.dev/projects
