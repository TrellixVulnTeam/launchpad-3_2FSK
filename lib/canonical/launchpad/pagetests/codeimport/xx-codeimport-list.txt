There is a page listing all the active code imports on the code
homepage.

As there are no *active* imports in the sampled data, we will mark the
existing three imports as active.

    >>> from canonical.database.sqlbase import flush_database_updates
    >>> from canonical.launchpad.ftests import login, logout
    >>> from canonical.launchpad.interfaces import (
    ...     ImportStatus, IProductSeriesSet)
    >>> from zope.component import getUtility
    >>> login('david.allouche@canonical.com')
    >>> product_series_set = getUtility(IProductSeriesSet)
    >>> for ps in product_series_set.searchImports():
    ...     ps.enableAutoSync()
    >>> flush_database_updates()
    >>> len(list(product_series_set.searchImports(
    ...     importstatus=ImportStatus.SYNCING)))
    3
    >>> logout()

The page is linked to from the "$N imported branches" text.

    >>> browser.open('http://code.launchpad.dev')
    >>> browser.getLink('3 imported branches').click()
    >>> print browser.title
    Available code imports

It lists the active imports, sorted by product then branch name:

   >>> def print_import_table():
   ...     table = first_tag_by_class(browser.contents, 'listing')
   ...     print extract_text(table)

   >>> print_import_table()
   Project   Series
         Source Details   Last Updated
   a52dec    failedbranch
         :pserver:anonymous@cvs.sourceforge.net:/cvsroot/liba52 a52dec
   a52dec     trunk
         :pserver:anonymous@cvs.sourceforge.net:/cvsroot/liba52 a52dec
   Evolution  trunk
         :pserver:anonymous@anoncvs.gnome.org:/cvs/gnome evolution

You can filter the list by product:

   >>> browser.getControl(name='text').value = 'a52dec'
   >>> browser.getControl('Search').click()
   >>> print_import_table()
   Project   Series
         Source Details   Last Updated
   a52dec    failedbranch
         :pserver:anonymous@cvs.sourceforge.net:/cvsroot/liba52 a52dec
   a52dec     trunk
         :pserver:anonymous@cvs.sourceforge.net:/cvsroot/liba52 a52dec
