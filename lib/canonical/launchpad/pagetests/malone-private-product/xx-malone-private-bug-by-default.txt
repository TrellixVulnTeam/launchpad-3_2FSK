Selecting the private_bugs checkbox on a product's +review page
will force bugs reported on that product to be private. (We will
demonstrate using the advanced filebug form, to skip searching for
dupes.)

    >>> from zope.component import getUtility
    >>> from canonical.launchpad.ftests import login, logout
    >>> from canonical.launchpad.interfaces import IProductSet

    >>> admin_browser.open("http://launchpad.dev/redfish/+review")
    >>> admin_browser.getControl(name="field.private_bugs").value = True
    >>> admin_browser.getControl("Change").click()

    >>> from zope.testbrowser.testing import Browser
    >>> browser = Browser()
    >>> browser.handleErrors = False

    >>> browser.addHeader("Authorization", "Basic foo.bar@canonical.com:test")
    >>> browser.open(
    ...     "http://launchpad.dev/redfish/+filebug-advanced")

    >>> browser.getControl(name="field.title").value = "a test summary"
    >>> browser.getControl(name="field.comment").value = (
    ...     "a test description")
    >>> browser.getControl("Submit Bug Report").click()

    >>> login("foo.bar@canonical.com")

    >>> redfish = getUtility(IProductSet).getByName("redfish")
    >>> redfish_bugtask = redfish.all_bugtasks[0]

    >>> redfish_bugtask.bug.private
    True

But not forced to be security-related, of course.

    >>> redfish_bugtask.bug.security_related
    False

Additionally, the owner is subscribed.

    >>> redfish.owner.name
    u'name12'

    >>> sorted(sub.name for sub in redfish_bugtask.bug.getDirectSubscribers())
    [u'name12', u'name16']

    >>> logout()
