Pocket for snap builds

Package repositories may contain many versions of a package, each in a different subdirectory or pocket.

If unsure, choose Updates.

Release
The current stable Ubuntu release. Package versions are frozen and do not change.
Security
Patches for security vulnerabilities in Ubuntu packages. They are managed by the Ubuntu Security Team and are designed to change the behavior of the package as little as possible -- in fact, the minimum required to resolve the security problem. As a result, they tend to be very low-risk to apply and all users are urged to apply security updates. Packages in Security are built with dependencies from Release and Security.
Updates
Updates for serious bugs in Ubuntu packaging that do not affect the security of the system. Packages in Updates are copied from Proposed once they have been tested.
Proposed
The testing area for updates. This repository is recommended only to those interested in helping to test updates and provide feedback. Packages in Proposed are built with dependencies from Release, Security, Updates and Proposed.
Backports
As the name states, these are unsupported new versions of packages which have been backported to an older release. Packages may contain new features, may introduce new interfaces, and bugs. Packages in Backports are built with dependencies from Release, Security and Backports.
