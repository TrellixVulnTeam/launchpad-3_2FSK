Using work items

Often, it can take a few separate steps to complete the work described in a blueprint. Launchpad lets you track these steps in the "Work items" text box.

Describing work items

It's easy to track the steps, or work items, necessary to complete the blueprint. Using the Work items text box, give a short description of each work item along with its status. For example:

      Design the UI: DONE
      Test the UI: TODO
      Bootstrap the dev environment: POSTPONED

Each work item goes on its own line, followed by a colon and its status.

Work item statuses

Each work item can have one of four statuses:

  • TODO
  • INPROGRESS
  • DONE
  • POSTPONED
