From the distrorelease task page, you can access the backport fix screen
to select specific releases to which a bugfix should be backported.

  >>> print http(r"""
  ... GET /distros/ubuntu/grumpy/+source/linux-source-2.6.15/+bug/10/+backport HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...

To backport to a specific release, select the checkbox beside that
release.

  >>> print http(r"""
  ... POST /distros/ubuntu/grumpy/+source/linux-source-2.6.15/+bug/10/+backport HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... Content-Type: application/x-www-form-urlencoded
  ... 
  ... target=warty.linux-source-2.6.15&savetargets=Target+Fix+to+Selected+Releases""")
  HTTP/1.1 303 See Other
  ...

You are brought back to the task page. The new bug task is shown in the
"also reported in" portlet.

  >>> print http(r"""
  ... GET /distros/ubuntu/grumpy/+source/linux-source-2.6.15/+bug/10/+editstatus HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...
  ...Also reported in...
  ...linux-source-2.6.15 (Ubuntu Warty)...
  ...
