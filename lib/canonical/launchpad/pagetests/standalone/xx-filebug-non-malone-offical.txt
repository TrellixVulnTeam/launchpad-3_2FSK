If you try to file a bug on a product which doesn't use Malone as its
bug tracker, you will be told so.

  >>> print http(r"""
  ... GET /products/alsa-utils/+filebug HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...
  ...alsa-utils does not use Malone as its bug tracker.
  ...
  ...To report a bug on alsa-utils,...
  ...

The same if you try to file a bug on a distribution which doesn't use
Malone.

  >>> print http(r"""
  ... GET /distros/debian/+filebug HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...
  ...Debian does not use Malone as its bug tracker.
  ...
  ...To report a bug on Debian, please do so in its official bug
  tracker or on its package in the distribution you're using.
  ...

It's also not possible to file a bug on any of the distribution's
source package.

  >>> print http(r"""
  ... GET /distros/debian/+source/mozilla-firefox/+filebug HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...
  Debian does not use Malone as its bug tracker.
  ...
  To report a bug about Debian, please do so in its official bug
  tracker or on its package in the distribution you're using.
  ...
