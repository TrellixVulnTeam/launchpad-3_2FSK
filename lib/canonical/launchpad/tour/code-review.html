Code review

Launchpad encourages high quality contributions to your project by making code review public. Anyone can propose their code to be merged into any other branch linked to your project. From there, your community can publicly discuss the proposed branch and vote on whether it’s right for inclusion.

Reviews by email and web

Code reviews turns merge proposals into mini-mailing lists.

Launchpad emails each comment and vote to everyone taking part in the review. It also maintains public visibility, making it easy for new participants to join in, with a web interface to the review.

Public voting

Public voting helps squash bugs before they make it into your code-base by increasing the number of people who can have their say.

When it comes to making a final decision on whether to accept someone’s branch, though, you naturally give more weight to the opinions of certain people. Your project’s community can help by voting on whether to accept or reject the proposed code. Using custom tags, each individual can even specify which aspect of the branch they’re voting on.
