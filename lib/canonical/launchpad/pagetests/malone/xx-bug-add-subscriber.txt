Non-Personal Subscriptions
==========================

In addition to subscribing / unsubscribing yourself, it is possible to add a
person as a subscriber to the bug. You would use this feature to subscribe
someone else to the bug.

Anonymous users cannot subscribe someone else to a bug.

    >>> anon_browser.handleErrors = True
    >>> anon_browser.open('http://bugs.launchpad.dev/firefox/+bug/1')
    >>> anon_browser.getLink('Subscribe someone else').click()
    >>> anon_browser.url
    'http://bugs.launchpad.dev/firefox/+bug/1/+addsubscriber/+login'

Users who are logged in can subscribe someone else to a bug.

    >>> user_browser.open('http://bugs.launchpad.dev/firefox/+bug/1')
    >>> user_browser.getLink('Subscribe someone else').click()
    >>> user_browser.url
    'http://bugs.launchpad.dev/firefox/+bug/1/+addsubscriber'

Make sure the other user is not already subscribed.

    >>> subs = find_portlet(user_browser.contents, 'Subscribers to bug 1')
    >>> 'No Privileges Person' not in subs('ul')[0].renderContents()
    True

Subscribe the other user to this bug.

    >>> user_browser.getControl("Person").value = 'no-priv'
    >>> user_browser.getControl("Add").click()
    >>> user_browser.url
    'http://bugs.launchpad.dev/firefox/+bug/1'

The user is notified that the other user has been subscribed.

    >>> for tag in find_tags_by_class(user_browser.contents,
    ...     'informational message'):
    ...     tag.renderContents()
    'No Privileges Person has been subscribed to this bug.'

The other user is listed as a subscriber to this bug.

    >>> subs = find_portlet(user_browser.contents, 'Subscribers to bug 1')
    >>> subs('ul')[0].renderContents()
    '...No Privileges Person...'
