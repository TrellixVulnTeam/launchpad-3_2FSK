
"Sample Person" has a calendar, which is displayed as a portlet:

  >>> print http(r"""
  ... GET /people/name12 HTTP/1.1
  ... """)
  HTTP/1.1 200 Ok
  Content-Length: ...
  Content-Type: text/html;charset=utf-8
  <BLANKLINE>
  ...
      <title>Sample Person Personal Information</title>
  ...
      <div class="portlet">
    <h4>Sample Person's Calendar</h4>
  ...


If we are logged in when viewing the page, A link is provided that
allows us to subscribe to the calendar:

  >>> print http(r"""
  ... GET /people/name12 HTTP/1.1
  ... Authorization: Basic Zm9vLmJhckBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  Content-Length: ...
  Content-Type: text/html;charset=utf-8
  <BLANKLINE>
  ...
      <div class="portlet">
    <h4>Sample Person's Calendar</h4>
  ...
        <p>
          <a href="+calendar/+subscribe">Subscribe to this calendar</a>
        </p>
  ...


The Mozilla project has no calendar:

  >>> print http(r"""
  ... GET /projects/mozilla HTTP/1.1
  ... """)
  HTTP/1.1 200 Ok
  Content-Length: ...
  Content-Type: text/html;charset=utf-8
  <BLANKLINE>
  ...
      <title>Upstream Project: The Mozilla Project</title>
  ...
    <!-- No calendar -->
  ...


However, if we are logged in as "Sample Person", then we have the
option to create a calendar for the project, since we have
launchpad.Edit permission on the project:

  >>> print http(r"""
  ... GET /projects/mozilla HTTP/1.1
  ... Authorization: Basic dGVzdEBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  Content-Length: ...
  Content-Type: text/html;charset=utf-8
  <BLANKLINE>
  ...
      <title>Upstream Project: The Mozilla Project</title>
  ...
    <!-- No calendar -->
    <div class="portlet">
      <h4>Calendar</h4>
      <div class="portletBody">
        <form action="+calendar-create" method="post">
          <p align="center">
            <input type="submit" value="Create Calendar" />
          </p>
        </form>
      </div>
    </div>
  ...
