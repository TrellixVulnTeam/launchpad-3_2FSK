IPOTemplate objects have IProductSeries or ISourcePackage as its direct
structural object.

Taking an IPOTemplate for IProductSeries context:

  >>> browser.open(
  ...     'http://translations.launchpad.dev/evolution/trunk/+pots/'
  ...     'evolution-2.2')
  >>> print browser.url
  http://translations.launchpad.dev/evolution/trunk/+pots/evolution-2.2

The Application tabs should point to IProductSeries URLs.

  >>> for tag in find_tags_by_class(browser.contents, 'tab-overview'):
  ...     print tag
  <...
  ...http://launchpad.dev/evolution/trunk/"...
  >>> for tag in find_tags_by_class(browser.contents, 'tab-branches'):
  ...     print tag
  <...
  ...unavailable tab-branches...
  >>> for tag in find_tags_by_class(browser.contents, 'tab-bugs'):
  ...     print tag
  <...
  ...http://bugs.launchpad.dev/evolution/trunk/"...
  >>> for tag in find_tags_by_class(browser.contents, 'tab-specifications'):
  ...     print tag
  <...
  ...http://blueprints.launchpad.dev/evolution/trunk/"...
  >>> for tag in find_tags_by_class(browser.contents, 'tab-translation'):
  ...     print tag
  >>> for tag in find_tags_by_class(browser.contents, 'tab-answers'):
  ...     print tag
  <...
  ...unavailable tab-answers...

But the actions should point to IPOTemplate urls.

  >>> print find_portlet(browser.contents, 'Actions').renderContents()
  <...
  ...http://translations.../evolution/trunk/+pots/evolution-2.2/"...Show translation status...
  ...http://translations.../evolution/trunk/+pots/evolution-2.2/+upload"...Upload a file...
  ...http://translations.../evolution/trunk/+pots/evolution-2.2/+export"...Download translations...
  ...http://translations.../evolution/trunk/+pots/evolution-2.2/+edit"...Change details...
